思想

快排使用了分治算法：将数组划分为两部分，然后对他们进行递归排序。例如，图1的八元组：



图1

我们围绕第一个元素(55)进行划分，使得所有小于55的元素都在它的左边，所有大于55的元素都在它的右边，如图2：

图2

于是数组被划分为两部分，然后对每个部分分别进行递归排序，最终即可得到一个有序的数组。


# 第一种代码实现



sort()方法描述了快排的基本框架，其中partitoin()的实现方式是影响快排效率的关键。图3表明了一种简单的实现方式：


图3

a,b分别是数组的第一个元素和最后一个元素的下标，t是划分值，m是在数组值小于t部分的最后一个元素的下标，i表示第一个待排序的元素的下标。

代码在检查第i个元素时需要考虑两种情况：当arr[i] >= t时, i直接向下一个元素移动；当arr[i]<t时，m移向下一个元素，然后交换arr[m]和arr[i]。

当循环终止时，有如下结果：



图4

则在m指向元素的左边都是小于t的，邮边都是大于t的。

优点：编码简单，易于理解；在随机排列中排序效率较高
缺点：当数组中所有元素的值相同时或者为升序，需要n-1次划分，算法效率退化至O(n2)

进一步改进

  1. 为了避免某些特殊情况的元素排列导致的性能降低，可以随机的选取数组中的某一个元素作为标准。
  2. 考虑到插入排序对于几乎有序的元素排列效率很高，可以对快排设置一个阈值y，即把sort()函数的第一个if语句改为if (hi - lo >= y) return;。这样，完成快排之后会剩下许多待排序的小数组，最后调用一次插入排序即可。
