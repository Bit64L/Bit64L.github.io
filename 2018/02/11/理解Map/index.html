Map的简单实现方式

采用二维数组pairs记录键值对, index记录数组被占用的大小。代码实现了Map的get()和put()方法，覆盖toString()方法用来输出map 的内容。

该实现方式的get()方法执行线性搜索时效率低下，HashMap利用散列码实现对键的快速查询。

使用HashMap可能遇到的问题

使用Integer, String等Java标准类库中类时，由于它们具备键需要的全部性质，Hashmap可以正常执行。但是当使用我们自己创建的类作为Hashmap的键时，可能会出现问题。这是因为我们需要重写Obejct类的hashcode()和equals()。

简单的散列Map

为了速度而散列: 映射表的瓶颈主要在键的查询速度。存储一组数据最快的方式为数组，因此可以通过键对象生成一个数字作为数组的下标，这个数字就叫做散列码。

  • 由于要自己实现散列Map就必须实现MapEntry类用来存取键值对，当键和值均相等时equals()方法返回true,hashcode()方法根据键和值生成散列码。SimpleHashMap为实现的简单散列Map。

  • SimpleHashMap使用桶作为数据结构，为了使散列分布均匀，桶的大小通常为质数。桶的示意图如下：

    关于桶的大小取质数：假设桶的大小为合数 z = x * y, 如果散列码为x的倍数，则散列范围会缩短到0 ~ y-1

  • 在SimpleHashMap中, put()方法根据键的hashcode进行散列，由于数组的大小有限并且索引为非负数，将hashcode取正然后取余。如果键已经存在，将原值赋给oldValue，新值替换旧值。

  • 在SimpleHashMap中, get()方法根据键的hashcode定位桶的位置，然后进行线性搜索Key，如果找不到，则返回null。

  • 在SimpleHashMap中, 由于entrySet()方法的返回值为Set<Entry<K, V>>，必须返回Set<Entry<K, V>>类型，非Set<MapEntry<K, V>>。

如何实现Hashcode()

基本指导

  1. 给int类型的result变量赋值一个常量
  2. 为对象内每个有意义的域f计算一个int散列码c
  3. 计算 result = 37 * result + c
  4. 返回result
  5. 检查hashcode()生成的结果，确保相同的对象具有相同的hashcode

示例

import java.util.LinkedList;

public class CountedString {

    private static LinkedList<String> list = new LinkedList<>();
    private String s;
    private int id;

    public CountedString(String str){
        s = str;
        list.add(s);
        for(String s1 : list){
            if(s1.equals(s)){
                id++;
            }
        }
    }

    @Override
    public String toString() {
        return "String:"+s+","+"id:"+id+","+"hashcode:"+s.hashCode();
    }

    @Override
    public boolean equals(Object obj) {
        return (obj instanceof CountedString) && (id == ((CountedString)obj).id)
                && (s.equals(((CountedString)obj).s));
    }

    @Override
    public int hashCode() {
        int result = 17;
        result = 37 * result + s.hashCode();
        result = 37 * result + id;
        return result;
    }
}/*Output
{String:h1,id:1,hashcode:144375=0, String:h1,id:4,hashcode:144378=3, String:h1,id:5,hashcode:144379=4, String:h1,id:2,hashcode:144376=1, String:h1,id:3,hashcode:144377=2}
Looking up String:h1,id:1,hashcode:144375
0
Looking up String:h1,id:2,hashcode:144376
1
Looking up String:h1,id:3,hashcode:144377
2
Looking up String:h1,id:4,hashcode:144378
3
Looking up String:h1,id:5,hashcode:144379
4
*/
  • 所有CountedString实例的字符串对存储在list中，id记录在list中与s相同的字符串的编号。
  • hashcode()和equals() 均根据s和id来生成结果，因此s相同id不同，产生的hashcode不同，对象也不相同。
